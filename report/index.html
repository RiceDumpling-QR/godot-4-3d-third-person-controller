01 · Experimental design

One repo, isolated evidence.

Branches separate the task seen by the agent from the hidden evaluation materials. Every run begins from the same ablated commit, while the verifier branch retains the working reference and behavioral tests.

Three independent copies of the ablated game. Each records one agent attempt under identical conditions.

02 · Behavioral delta

The break is visible.

The camera, combat, world, HUD, and character remain intact in both builds. The controlled difference is locomotion: the original traverses the level; the ablated build cannot translate or jump.

03 · Feature selection

Why ablate locomotion?

Locomotion is central enough to matter, broad enough to expose engineering judgment, and concrete enough to grade from behavior rather than code shape.

01 / Player critical

Immediate experiential impact

Removing locomotion makes a polished game unmistakably incomplete without destroying its surrounding systems. A successful repair is obvious to a player.

02 / Integrative

More than WASD

A complete fix coordinates camera-relative input, acceleration, collision, grounded jumping, variable jump height, facing, animation, footsteps, and landing feedback.

03 / Observable

Implementation-agnostic grading

The verifier measures displacement, velocity, direction, orientation, jump trajectory, and landing state. Agents can choose their own code structure.
